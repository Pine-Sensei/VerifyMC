# VerifyMC 系统全面审查 Checklist

## 后端架构审查
- [x] VerifyMC.java 主类职责是否单一，是否有过多的依赖和功能
  - **结果**: ❌ 职责过重，承担了生命周期管理、命令处理、事件监听、白名单同步、数据迁移等多种职责
- [x] 服务层（Service 类）是否正确实现了单一职责原则
  - **结果**: ⚠️ 部分服务类职责不清，如 AuthmeService 同时处理数据库连接、密码加密、用户同步
- [x] DAO 层是否正确抽象了数据访问逻辑
  - **结果**: ✅ DAO 层设计合理，接口抽象良好
- [x] Web 处理器（Handler 类）是否正确处理了 HTTP 请求
  - **结果**: ⚠️ WebServer.java 过于庞大（1862行），路由定义和业务逻辑混在一起
- [x] 模块间耦合度是否合理，是否存在循环依赖
  - **结果**: ⚠️ 缺乏依赖注入，使用 Setter 注入导致对象可能在未完全初始化状态下被使用
- [x] 是否正确使用了依赖注入或控制反转
  - **结果**: ❌ 没有使用依赖注入框架

## 安全性审查
- [x] 用户名验证是否完整（长度、格式、特殊字符）
  - **结果**: ✅ 使用正则表达式验证，支持自定义规则
- [x] 邮箱验证是否完整（格式、域名白名单）
  - **结果**: ⚠️ 正则表达式过于宽松，允许无效格式
- [x] UUID 验证是否完整（格式、有效性）
  - **结果**: ✅ 使用正则表达式验证 UUID 格式
- [x] SQL 查询是否使用了参数化查询，防止 SQL 注入
  - **结果**: ✅ 大部分使用 PreparedStatement，但存在配置值拼接风险
- [x] 用户输入是否在输出前进行了适当的编码，防止 XSS
  - **结果**: ✅ 实现了 escapeHtml 方法
- [x] 密码是否使用了安全的哈希算法存储
  - **结果**: ⚠️ 使用 SHA-256 加盐，建议使用 BCrypt 或 Argon2
- [x] 会话令牌是否使用了安全的随机生成
  - **结果**: ❌ 使用 Math.random()，不安全
- [x] 会话令牌是否有适当的过期机制
  - **结果**: ✅ 有过期机制（5分钟）
- [x] 管理员 API 是否正确验证了认证状态
  - **结果**: ✅ 使用 webAuthHelper.isAuthenticated() 验证
- [x] 敏感信息（密码、API 密钥）是否不会出现在日志中
  - **结果**: ⚠️ 调试日志可能包含敏感信息
- [x] Discord OAuth2 实现是否安全（state 参数验证）
  - **结果**: ✅ 正确验证 state 参数
- [x] 验证码机制是否安全（防重放攻击）
  - **结果**: ❌ 使用 java.util.Random，可预测
- [x] 问卷提交是否有防篡改机制
  - **结果**: ✅ 使用 token 验证和过期机制

## 性能审查
- [x] 数据库查询是否使用了索引
  - **结果**: ⚠️ 缺少 status、discord_id、regTime 索引
- [x] 是否存在 N+1 查询问题
  - **结果**: ✅ 未发现 N+1 问题
- [x] 是否有不必要的全表扫描
  - **结果**: ⚠️ LIKE '%keyword%' 查询导致全表扫描
- [x] 内存中的数据结构是否有大小限制
  - **结果**: ❌ languageCache、questionnaireSubmissionStore 无大小限制
- [x] 是否有潜在的资源泄漏（连接、流等）
  - **结果**: ⚠️ 数据库连接未使用连接池
- [x] 并发操作是否线程安全
  - **结果**: ⚠️ FileUserDao 同步锁粒度过大
- [x] 是否正确使用了异步处理
  - **结果**: ❌ 直接创建线程而非使用线程池
- [x] WebSocket 连接是否有适当的资源管理
  - **结果**: ⚠️ 缺少心跳和连接数限制
- [x] 问卷提交存储是否有清理机制
  - **结果**: ❌ 无自动清理机制
